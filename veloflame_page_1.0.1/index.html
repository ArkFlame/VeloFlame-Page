Features

Connection security before backend load.

VeloFlame focuses on the proxy edge: the point where bad traffic can be rejected before Paper, Spigot, Folia, login plugins, and databases do unnecessary work.

Proxy firewall

Block abusive IPs and connection patterns at the proxy layer with configurable activation thresholds and logs.

Connection checks

Use rate limits, reconnect controls, VPN/proxy lists, and allowlists to reduce low-quality traffic.

Backend relief

Rejecting obvious abuse earlier helps backend servers stay focused on gameplay instead of handshake noise.

Performance discipline

Cut work per bad connection.

The highest-return proxy optimization is not decorative. It is doing less work for traffic that should never reach the backend. VeloFlame keeps those controls visible in configuration.
